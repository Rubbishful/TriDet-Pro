# 基于 TriDet 的时序动作检测课程设计 — 重构开发规划

## Context

本项目是 CVPR 2023 TriDet 论文的代码实现与改进。当前 THUMOS14 基线已复现（mAP 68.59%，论文 69.27%）。

**原规划的核心问题**：将 TriDet 视为黑盒，仅在外围添加可视化、Subject 检测（YOLOv8n+SORT）、端到端特征提取（X3D-S），完全没有涉及多实例检测与重叠动作处理——这是 TAD 领域的核心挑战。

**本重构规划的核心思路**：从根本上改进 TriDet 的检测能力，优先解决多实例检测与重叠动作处理问题，在 TriDet 已有的 SGP 时序建模基础上增强，而非重新发明。

### TriDet 架构根本局限

问题根源在 [libs/modeling/meta_archs.py](libs/modeling/meta_archs.py) `label_points_single_video` 方法（L571-652）：每个时序特征点只能被分配到一个 GT 动作实例（通过最短时长打破平局）。这导致：
- **同类别多实例**：两个 "BasketballDunk" 在时间上接近时，竞争相同的特征点，其中一个可能被遗漏
- **异类别重叠动作**：虽然 per-class NMS 理论上支持，但共享特征表示只从单个 GT 段学习，判别力不足
- **分类为多标签（sigmoid），但回归为单目标**：架构不对称

## Phase 1: 深度错误分析（2 周）

**目标**: 量化 TriDet 在多实例冲突、重叠动作、不同时长/密度场景下的失败模式，为 Phase 2-4 提供数据驱动指导。

**分析维度**:
1. Per-class mAP 分解
2. 按时长分层（短 <2s / 中 2-6s / 长 >6s）的检测性能
3. 按视频密度（动作实例数）分层的性能退化
4. **多实例冲突量化**（核心指标）：同类别时间接近的实例对数量及其 recall
5. **重叠动作分析**：异类别重叠段的双重检测成功率
6. 边界预测误差分布（起止边界 MAE，与时长相关性）

**新建文件**:
- `analysis/` 模块（`error_analyzer.py`, `multi_instance_analyzer.py`, `overlap_analyzer.py`, `boundary_analyzer.py`, `visualize_analysis.py`）

**修改文件**:
- `libs/utils/metrics.py` — 返回 per-class AP

**验证**: 生成完整分析报告，关键输出为冲突实例 vs 非冲突实例 mAP 差异（预期 >10%）

---

## Phase 2: 多实例检测改进（3 周）

**目标**: 解决同类别多实例在时间上接近时的漏检问题。

**推荐路线：MI-QLA（Multi-Instance Quality-aware Label Assignment）**

修改 `label_points_single_video` 的标签分配逻辑：
- 每点不只选最短 GT，而是找到所有匹配 GT，按质量分数（IoU 贡献度）加权
- 分类目标：多个 GT 的 one-hot 加权求和
- 回归目标：质量加权平均偏移量
- 引入质量加权的回归损失（参考 Generalized Focal Loss, NeurIPS 2020）

**备选路线：Query-based 检测分支**

基于 PointTAD/DualDETR 思想，在 SGP 特征上添加少量 learnable queries（~30 个），Hungarian matching 一对一匹配，与 dense head 并行推理合并。

**参考论文**: PointTAD (NeurIPS 2022), DualDETR (CVPR 2024), Generalized Focal Loss (NeurIPS 2020)

**核心修改**:
- `libs/modeling/meta_archs.py` — 修改标签分配逻辑
- `libs/modeling/losses.py` — 添加质量加权回归损失

**验证**: 多实例冲突场景 recall 相对提升 >15%，非冲突场景性能不退化，显存增加 <500MB

---

## Phase 3: 重叠动作处理（2.5 周）

**目标**: 增强特征表示以区分时间上重叠的不同类别动作。

**技术方案**:

**3a. Per-class 特征解耦模块** — 在分类头前插入轻量通道注意力，增强类别维度的可分离性

**3b. 双路径检测架构**（参考 ActionSwitch ECCV 2024）:
- Boundary Path：大卷积核（k=7）增强边界检测
- Semantic Path：小卷积核（k=3）保持局部分类能力
- 共享 SGP 骨干，仅卷积核大小不同

**3c. 重叠感知损失函数** — 对重叠区域的分类损失加权

**参考论文**: ActionSwitch (ECCV 2024), RefDense (arXiv 2025), ReAct (ECCV 2022)

**核心修改**:
- `libs/modeling/blocks.py` — 新增 `FeatureDisentangleModule`
- `libs/modeling/meta_archs.py` — 集成双路径逻辑
- `libs/modeling/losses.py` — 添加 `overlap_aware_loss`

**验证**: 重叠区域双重检测成功率显著提升

---

## Phase 4: 时序建模增强（2 周）

**目标**: 在 SGP 已有基础上增强，而非重新实现。

TriDet 的 SGP 已提供了 O(T) 的 Transformer 级时序建模。本阶段做增量增强：

**4a. Dynamic-SGP**（参考 DyFADet ECCV 2024）:
- 在 `SGPBlock` 中添加动态卷积核生成器（轻量 1x1 conv + AdaptiveAvgPool）
- 根据输入内容自适应调整感受野
- 参数增加约 0.3M

**4b. 多尺度特征对齐（MSFA）**:
- 在 FPN top-down path 中插入可变形偏移对齐模块
- 改善不同 FPN 层级间的特征融合

**4c. 时序关系模块（TRM）**（参考 ReAct ECCV 2022）:
- 仅在 FPN 最高分辨率层（stride=1, T<400）上使用轻量自注意力
- 建模不同时间位置的动作间关系

**核心修改**:
- `libs/modeling/blocks.py` — Dynamic-SGP + TRM
- `libs/modeling/necks.py` — MSFA

**验证**: 每个子模块独立消融，显存增加各 <200MB

**5a. 消融矩阵**（6 组实验：基线 → 逐模块叠加）

**新建文件**:

    │   ├── Phase 3 (重叠动作) ──── 依赖 Phase 2

## 与原规划对比

| 维度 | 原规划 | 新规划 |
|------|--------|--------|
| 核心目标 | TriDet 黑盒包装 | TriDet 检测能力根本性改进 |
| 多实例检测 | 未涉及 | **Phase 2 核心内容** |
| 重叠动作 | 未涉及 | **Phase 3 核心内容** |
| 时序建模 | 误认为 TriDet 缺失此功能 | **Phase 4 在 SGP 基础上增量增强** |
| 可视化 | 独立阶段 | 集成在 Phase 5/6 分析中 |
| Subject 检测 | 独立阶段 | 降级为可选探索方向 |
| 端到端特征提取 | 独立阶段 | 降级为可选探索方向 |
| 错误分析 | 嵌入 Phase 5 | **Phase 1 先行，数据驱动后续决策** |
| 技术深度 | 调用现成库 (YOLO, SORT, X3D) | 深入修改 TriDet 核心架构 |

## 风险评估

| 风险 | 概率 | 影响 | 应对 |
|------|:---:|------|------|
| MI-QLA 导致 mAP 退化 | 中 | 高 | `mi_quality_weight` 可调；备选 query 分支 |
| 多模块叠加显存溢出 | 低 | 高 | 每模块参数增量 <0.5M；模块可开关 |
| ActivityNet 训练时间过长 | 中 | 中 | Phase 5 仅训最优配置 |
| 学期时间不够 | 中 | 中 | Phase 4 可缩减为仅 Dynamic-SGP |

## 验证方法

1. **Phase 1**: 分析脚本生成完整报告，量化冲突实例 mAP 差距

4. **Phase 4**: 每个子模块独立消融验证，显存增加可控
