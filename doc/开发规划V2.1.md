## Phase 2: 多实例 / 重叠动作 / 主体检测（第 1 周，4-5 天）

**核心思路**: 分析为主导 + 轻量后处理改进 + 即用型模型集成。不深入修改 TriDet 训练流程。

### 2a. 多实例与重叠动作分析（1-2 天）

**做什么**: 分析 TriDet 在密集/重叠场景的表现，找到具体失效案例，提出简单改进。

**分析脚本** (`analysis/error_analysis.py`):
- 统计每个视频的动作实例数量，按密度分层计算 recall
- 找出同类别时间接近（gap < 3s）的实例对，计算基线同时检出率
- 在 THUMOS14 验证集上生成 5-10 个典型失效案例的时间轴图

**简单改进 — Density-Aware NMS**（修改 `libs/utils/nms.py`）:
- 同类 NMS 阈值降低（从 soft-NMS sigma=0.75 改为 0.85），让同类多实例更容易共存
- 或：根据局部预测密度动态调整 NMS 阈值，密度高处更宽松
- 这是 ~20 行修改，不影响训练，只改推理后处理

**预期效果**: 多实例场景 recall 提升 5-10%，整体 mAP 不退化或微涨

### 2b. 主体（人物）检测关联（1-2 天）

**做什么**: 对检测到的动作实例，关联视频中的人物主体。

**技术路线**（极简版）:
- 使用 `ultralytics` 库的 YOLOv8n（预训练，无需自己训练）
- 在动作片段中间帧检测人物，取置信度最高的检测框
- 如果动作覆盖多帧，在起止帧各检测一次，取平均位置
- 关联逻辑：时间戳匹配（动作中间帧 → 最近的关键帧检测结果）

**新建文件**:
- `libs/subject/detector.py` — 封装 YOLOv8n 推理（~50 行）

**修改文件**:
- `eval.py` — 添加 `--with-subject` 选项，输出中附加 subject 信息

**验证**: 选 5 个 THUMOS14 体育动作视频，可视化检测框+动作区间

### 2c. 重叠动作可视化增强（0.5-1 天）

- 标注哪些区域存在多类别重叠

**估时**: 4-5 天

## Phase 3: 架构改进 / 端到端特征（第 2 周，4-5 天）

### 3a. 轻量架构改进（2-3 天）

**做什么**: 在 SGP 骨干中插入通道注意力模块（SE Block），增强特征判别力。

**为什么选 SE Block**:
- 实现极简（~15 行代码），插入到 `SGPBlock` 的残差连接后
- 不改变特征图尺寸，不影响下游所有模块
- 参数增量极小（2 * C^2 / reduction ≈ 几千参数）
- 是经过充分验证的通用改进，在分类/检测任务上普遍有效
- 本科生课设中易于解释（"让网络自动学习不同通道的重要性"）

**实现** (`libs/modeling/blocks.py`):
```python
class SELayer(nn.Module):
    def __init__(self, channels, reduction=16):
        super().__init__()
        self.fc = nn.Sequential(
            nn.AdaptiveAvgPool1d(1),
            nn.Conv1d(channels, channels // reduction, 1),
            nn.ReLU(),
            nn.Conv1d(channels // reduction, channels, 1),
            nn.Sigmoid()
        )
    def forward(self, x):
        return x * self.fc(x)
```

**修改**:
- `libs/modeling/blocks.py` — 添加 `SELayer`，在 `SGPBlock.forward()` 末尾添加通道注意力
- `configs/thumos_i3d_se.yaml` — 新配置（继承基线，启用 SE）

**训练+评估**: 在 THUMOS14 上训练 20 epoch，对比基线 mAP。目标：0.5-2% 提升。

**备选**（如果 SE 效果不好或时间充裕）:
- 调整 SGP 的多尺度卷积核参数 k（当前 THUMOS 用 5.0），尝试 k=3.0/7.0
- 或：添加 DropPath 正则化强度搜索

### 3b. 端到端特征提取 — 概念验证（2 天）

**做什么**: 用预训练 3D CNN 替代预提取的 I3D 特征，证明端到端流程可行。

**为什么是概念验证而非完整实现**:
- 完整端到端训练（原始视频 → 3D CNN → TriDet）显存不够（需 >24GB）
- 两步方案更实际：用预训练 3D CNN 离线提取特征 → 替代 I3D → 训练 TriDet
- 本科生课设展示"特征可替换性"即可，不需要联合训练

**技术路线**:
- 使用 `torchvision.models.video` 中的 S3D 或 X3D-S 预训练模型
- 对 THUMOS14 原始视频（需下载部分）提取特征，或直接在已下载的帧上运行
- 将提取的特征保存为 .npy，格式对齐现有 DataLoader
- 用新特征训练 TriDet，对比 I3D 特征的结果

**简化方案**（如果原始视频获取困难）:
- 不做实际替换，改为详细的方案设计文档 + 伪代码
- 分析显存/计算量预算、特征维度对齐、训练策略
- 在报告中作为"端到端扩展设计"章节

**新建文件**:
- `scripts/extract_features.py` — 3D CNN 特征提取脚本
- （可选）`configs/thumos_e2e.yaml` — 端到端配置

**验证**: 提取的特征维度与 I3D 格式一致；或用文档充分说明设计

**估时**: 4-5 天

## 展望（非 Phase，报告中的未来方向章节）

简单列举 3-5 个有技术价值的后续方向，每个方向 2-3 句话即可：

1. **端侧压缩部署**: 使用知识蒸馏或结构化剪枝将 TriDet 压缩至移动端可运行规模，结合 TensorRT/ONNX 推理优化，适用于安防摄像头等边缘场景的实时动作检测。
2. **国产 NPU 适配**: 将模型迁移到华为昇腾（Ascend）或寒武纪 MLU 平台，利用 Cann 或 MagicMind 工具链进行算子适配和混合精度推理。
3. **端到端视频理解**: 用 VideoMAE 等自监督预训练骨干替代 I3D 特征，实现从原始视频到检测结果的完整可微分流程，减少特征提取的任务偏差。
4. **开放集动作检测**: 结合 CLIP 等多模态模型，使检测器能识别训练时未见过的动作类别，提升零样本泛化能力。
5. **多模态融合**: 融合 RGB + 光流 + 音频特征，在复杂场景（遮挡、暗光）下提升检测鲁棒性。
