# 基于 TriDet 的时序动作检测课程设计 — 开发规划

## Context

本项目是 CVPR 2023 TriDet 论文的代码实现，需要在此基础上完成课程设计任务：
复现结果 + 性能提升 + Subject 检测 + 端到端处理 + 可视化 + 性能分析。

**约束**: GPU 约 16GB 显存，主数据集 THUMOS14（400 视频/20 类），扩展 ActivityNet。

---

## Phase 1: 环境搭建与基线复现

**目标**: 使用预提取 I3D 特征训练 TriDet，在 THUMOS14 上复现论文结果。

**关键步骤**:
1. 安装依赖 (`requirements.txt` + PyTorch >= 1.11)
2. 编译 NMS C 扩展 (`python libs/utils/setup.py build_ext --inplace`)
3. 下载 THUMOS14 数据集 + I3D 预提取特征
4. 训练: `python train.py ./configs/thumos_i3d.yaml --output baseline`
5. 评估: `python eval.py ./configs/thumos_i3d.yaml ./ckpt/baseline/`

**修改文件**: 无（验证现有代码可运行）

**验证**: mAP 达到 ~72.9% @ tIoU=0.5（与论文一致）

---

## Phase 2: 可视化模块

**目标**: 在时间轴上可视化 GT 和预测的动作区间、类别、置信度。

**新建文件**:
- `libs/visualization/__init__.py`
- `libs/visualization/visualizer.py` — `TADVisualizer` 类，使用 matplotlib 绘制
- `visualize.py` — 入口脚本

**核心设计**:
- 双行布局：上行 GT，下行预测
- 不同类别使用不同颜色（tab20 colormap）
- 预测条透明度编码置信度
- 支持单视频详细视图 + 批量导出

**修改文件**: 无

**验证**: 在几个视频上运行，肉眼检查时间轴是否正确

---

## Phase 3: Subject 检测模块

**目标**: 对每个检测到的动作识别执行主体（人物）。

**策略**（考虑了 16GB 约束）:
- 使用轻量 YOLOv8n (~3M 参数) 在 1 FPS 采样帧上检测人物
- SORT（Kalman + Hungarian）进行简单跟踪
- 时间关联：对每个动作区间，寻找时间重叠度最高的人物轨迹

**新建文件**:
- `libs/subject/__init__.py`
- `libs/subject/detector.py` — YOLO 封装
- `libs/subject/associator.py` — 关联逻辑

**修改文件**:
- `eval.py` — 添加 `--with-subject` 标志
- `libs/modeling/meta_archs.py` — 输出结果扩展 `subjects` 字段
- `libs/visualization/visualizer.py` — 可视化中加入 subject 信息

**验证**: 在示例视频上检查人物检测和关联是否合理

---

## Phase 4: 端到端特征提取器

**目标**: 用可学习的 3D CNN 替代预提取 I3D 特征，实现端到端训练。

**策略**（两阶段）:
- **Phase A**: 冻结 3D CNN，仅训练 TriDet，使用梯度检查点节省显存
- **Phase B**: 解冻 3D CNN 最后几层，联合微调（AMP 混合精度）

**选用 X3D-S**（~3M 参数），以 16 帧 mini-clip 滑动窗口提取特征。

**新建文件**:
- `libs/modeling/feature_extractors.py` — X3D 等特征提取器
- `libs/datasets/thumos14_raw.py` — 从原始视频加载帧的数据集
- `configs/thumos_e2e.yaml` — 端到端训练配置

**修改文件**:
- `libs/modeling/models.py` — 注册 feature_extractor 工厂函数
- `libs/modeling/meta_archs.py` — forward 中集成特征提取器

**显存预算**:
| 组件 | 显存 |
|------|------|
| TriDet (batch=2) | ~3 GB |
| X3D-S (batch=2, 16帧) | ~2 GB |
| 总计 (AMP) | ~8-10 GB |

**验证**: Phase A 的 mAP 接近 I3D 基线；Phase B 超过或接近 I3D 基线

---

## Phase 5: 性能提升实验与分析

**目标**: 超越基线性能 + 系统性分析不同因素对检测的影响。

### 5a. 提升实验（按优先级）

1. **训练策略**：增长训练 epoch、标签平滑、EMA 调参、输入噪声 — 低开销
2. **数据增强**：时序抖动、特征 Dropout、时序 Mixup — 修改 `libs/datasets/thumos14.py`
3. **架构调优**：调整 `backbone_arch`、`embd_dim`、SGP 窗口大小、启用 FPN — 修改 YAML 配置
4. **损失函数**：调整 `iou_weight_power`、GIoU vs DIoU — 修改 YAML 配置

### 5b. 分析模块

**新建文件**:
- `analysis/__init__.py`
- `analysis/performance_analyzer.py` — `PerformanceAnalyzer` 类
- `analysis/run_analysis.py` — 入口

**分析维度**:
- 按时长分解 mAP（短/中/长动作）
- 按视频复杂度（动作实例数量）
- Per-class 性能
- 边界预测误差分布

**验证**: 生成有统计意义的图表和表格

## 验证方法

1. **Phase 1**: mAP 与论文结果对比
2. **Phase 2**: 肉眼检查可视化输出图片
3. **Phase 3**: 在含人物的体育视频上验证 subject 关联
4. **Phase 4**: 对比端到端训练 vs I3D 预提取特征的 mAP 差异
5. **Phase 5**: 每项实验记录 mAP 变化；分析脚本输出统计图表
6. **Phase 6**: `python demo.py` 一键完成检测→可视化全流程
